Introdução¶

O presente estudo tem como objetivo Mensurar a Mobilidade Intergeracional Educacional no Brasil, ou seja, em que medida é possível que uma geração posterior ultrapasse a anterior. Seguindo a metodologia Rank-Rank foram realizadas análises entre o nível de escolaridade dos pais e as respectivas notas de seus filhos, são esperados fortes relações entre o nível de escolaridade dos pais e as notas dos alunos onde há menor mobilidade intergeracional, isto é, quanto mais determinante seja a escolaridade dos pais para que o aluno seja melhor rankeado em seu estado, segundo a nota do Enem.

Metodologia¶

A base de dados utilizada para este estudo contém uma amostra de 100.000 resultados de participantes do ENEM em 2017. A média de idade observada nesta amostra é de pouco mais de 22 anos, uma média relativamente alta, considerando que o exame é direcionado para alunos do último ano do ensino médio, além disso, 50% de nossa amostra apresenta idade superior a 20 anos, tendo como idade mínima e máxima, 14 e 77 anos respectivamente e desvio padrão de 7 anos, demonstrando que a idade dos participantes não estão muito dispersas da média.

As famílias dos participantes apresentam em média 4 membros, o que coloca a média de nossa amostra acima da média nacional que é 3 membros, segundo a POF. Quanto às notas, a menor nota de nossa amostra foi de 154.91 pontos e a maior foi de 818.14, sendo que a média foi de 520 pontos. Mais de 50% dos participantes de nossa base de dados apresentaram uma nota inferior ao média e 75% apresentaram nota inferior a 600 pontos.

# função que define os limites da unidade de medida
# data: dataframe
# group_by: coluna utilizada para agrupar
#                Caso preenchido o ranking será contabulizado tomando apenas as notas
#                obtidas por alunos de um mesmo grupo (sg_uf_residencia ou co_municipio_residencia)
#                essa opção fará uma _normalização_ das notas por localidade
# clusters: domínio do ranking
#
# return: Retorna os limites de cada cluster para cada grupo (se informado)
def group_fixed_range(data, group_by, clusters):
    # colunas do range
    col = [group_by]
    for c in clusters:
        col.append(c)
        
    # caso não seja informado o grupo, o ranking será realizado 
    # utilizando todo o dataset
    group = pd.DataFrame()
    if len(group_by) == 0:
        fixed_range = ['']
        _max = max(data[grade_column])         # maior nota do grupo
        _min = min(data[grade_column])         # menor nota do grupo
        _offset = (_max - _min) / len(clusters)
        for c in range(1,len(clusters)+1):
            fixed_range.append(_min + _offset * c)
        group = group.append(pd.DataFrame([fixed_range], columns=col), ignore_index=True)
        return group
    
    # recupera os valores a serem a grupados
    keys = data[group_by].unique()
    group = pd.DataFrame(columns=col)

    # realiza o cáculo do range de cada cluster para cada grupo
    for key in keys:
        df_key = data[data[group_by] == key]     # linhas de um mesmo grupo (key)
        
        fixed_range = [key]
        _max = max(df_key[grade_column])         # maior nota do grupo
        _min = min(df_key[grade_column])         # menor nota do grupo
        _offset = (_max - _min) / len(clusters)  # intervalo de cada cluster
        
        for c in range(1, len(clusters) + 1):
            fixed_range.append(_min + _offset * c)
        group = group.append(pd.DataFrame([fixed_range], columns=col), ignore_index=True)
    return group


# Rankea uma amostra a partir dos parâmetros
# data: dataframe
# group: grupo contendo os limites de cada cluster
# group_by: coluna utilizada para agrupar
#                Caso preenchido o ranking será contabulizado tomando apenas as notas
#                obtidas por alunos de um mesmo grupo (sg_uf_residencia ou co_municipio_residencia)
#               essa opção fará uma _normalização_ das notas por localidade
# clusters: domínio do ranking
#
# return: Retorna o valor em texto do ranking
def ranking(row, group, group_by, clusters):
    # verifica se a condição de agrupamento foi informada
    if len(group_by) >0:    
        g = group[group[group_by] == row[group_by]]
    else:
        g = group

    # itera entre os clusters
    for c in clusters:
        if (row[grade_column] <= g[c].values):
            return c
    return 'NaN'


# Rankea uma amostra a partir dos parâmetros
# data: dataframe
# group: grupo contendo os limites de cada cluster
# group_by: coluna utilizada para agrupar
#                Caso preenchido o ranking será contabulizado tomando apenas as notas
#                obtidas por alunos de um mesmo grupo (sg_uf_residencia ou co_municipio_residencia)
#                essa opção fará uma _normalização_ das notas por localidade
# clusters: domínio do ranking
#
# return: Retorna o índice do ranking
def ranking_index(row, group, group_by, clusters):
    # verifica se a condição de agrupamento foi informada
    if len(group_by) >0:
        g = group[group[group_by] == row[group_by]]
    else:
        g = group
    
    # itera entre os clusters
    for c in range(len(clusters)):
        if (row[grade_column] <= g[clusters[c]].values):
            return c + 1
    return -1


# Adiciona colunas ao dataset
# data: dataframe
# group_by: coluna utilizada para agrupar
#
# return: dataframe com novas colunas
def add_columns(data, group_by):
    # cria novas colunas para o dataset
    
    no_group_suffix = str(grade_column) + '_no_group'
    group_suffix = str(grade_column) + '_' + group_by
    
    _new_columns = [
                 'sq_renda',
                 'ranking_' + no_group_suffix, 
                 'sq_ranking_' + no_group_suffix,
                 'ranking_' + group_suffix, 
                 'sq_ranking_' + group_suffix,
                 'ranking_pais',
                 'sq_ranking_pais']

    data['sq_renda']= -1                           # índice da renda
    data['ranking_' + no_group_suffix] = ''        # ranking das notas sem agrupamento
    data['sq_ranking_' + no_group_suffix] = -1       # índice do ranking das notas sem agrupamento
    data['ranking_' + group_suffix] = ''           # ranking das notas com agrupamento
    data['sq_ranking_' + group_suffix] = -1        # índice do ranking das notas com agrupamento
    data['ranking_pais']= ''                       # novo ranking dos pais
    data['sq_ranking_pais']= ''                    # índice do ranking dos pais

    return data, _new_columns


# Recupera índice do rankig dos pais
# pai: ranking de educação do pai
# mae: ranking de educação da mãe
#
# return: retorna o maior nível de educação entre o pai/mãe
def ranking_pais(pai, mae):
    for c in reversed(range(len(educ_ranking))):
        if( pai == educ_ranking[c] or mae == educ_ranking[c]):
            return educ_ranking[c]


# Recupera índice do rankig dos pais
# pai: ranking de educação do pai
# mae: ranking de educação da mãe
#
# return: retorna o maior nível de educação entre o pai/mãe
def ranking_pais_index(pai, mae):
    for c in reversed(range(len(educ_ranking))):
        if( pai == educ_ranking[c] or mae == educ_ranking[c]):
            return (c + 1)

        
# Recupera índice do rankig da renda           
#
# return: retorna o ínidice da renda 
def ranking_renda_index(renda):
    for c in reversed(range(len(income_ranking))):
        if(renda == income_ranking[c]):
            return (c + 1)   
            


def eval_ranking(data, group_by, clusters):
    # recupera os grupos a partir da condição group_by
    group = group_fixed_range(data, group_by, clusters)
    # recupera os grupos sem condição group_by
    no_group = group_fixed_range(data, '', clusters)
    
    data, new_columns = add_columns(data, group_by)
    
    # itera em todas as linhas do dataset (dataframe)
    for index, row in data.iterrows():
        data.at[index, new_columns[0]] = ranking_renda_index(row['renda_fam']) # índice do ranking renda
        
        data.at[index, new_columns[1]] = ranking(row,no_group, '', clusters)        # ranking sem grupo
        data.at[index, new_columns[2]] = ranking_index(row,no_group, '', clusters)  # ínidice ranking sem grupo
        
        data.at[index, new_columns[3]] = ranking(row,group, group_by, clusters)       # ranking
        data.at[index, new_columns[4]] = ranking_index(row,group, group_by, clusters) # ínidice ranking
        
        data.at[index, new_columns[5]] = ranking_pais(row['educ_pai'], row['educ_mae'])       # ranking educ pais
        data.at[index, new_columns[6]] = ranking_pais_index(row['educ_pai'], row['educ_mae']) # indice ranking educ pais
    return data

Resultados¶

Nesta seção realizaremos a análise dos modelos estimados para medir o impacto da educação parental em relação ao desempenho do aluno no exame do Enem. Partindo do modelo inicial de regressão linear Rank-Rank definido por:

$$r_{i,j} = \alpha + \beta RP_i + e_{i,j}$$

Sendo $r_{i,j}$ é o ranking do aluno $i$ em seu estado $j$, $RP$ é o ranking do aluno $i$, $e_{i,j}$ é o erro. As estimativas $\alpha$ e $\beta$ indicarão o valor médio do ranking de cada aluno, independente do ranking dos pais, e o impacto da escolaridade dos pais nas notas dos exames dos filhos, respectivamente.

Modelo Rank-Rank¶

Ambas estimativas resultantes são significantes a 1% (P < 0,01). O Intercepto $\alpha$, indica a posição do ranking do participante independentemente da posição do ranking dos pais. Dessa forma, o desempenho médio dos participantes, sem levarmos em consideração o ranking dos pais, os colocam no nível "C". O coeficiente $\beta$, nos diz que em média, a variação de uma unidade de ranking dos pais, faz com que os participantes aumentem 0,2405 unidades de ranking na nota do Enem.

Estimativa Macroeconômica¶

Para o teste de correlação foram colhidos dados do FUNDEB transferido aos municípios, na Secretaria do Tesouro, do ano de 2007 a 2017. Para ter uma aproximação do número de estudantes, foram coletadas a quantidade de matrículas dos estudantes da educação básica a partir do Censo Escolar disponibilizado no INEP, entre os anos de 2007 e 2017. A média das transferências do FUNDEB e das matrículas foram efetuadas para que no fim fosse calculada a média dos recursos de transferências do FUNDEB por aluno. Em um primeiro momento foi calculado a correlação entre os coeficientes e a média das transferências por aluno, porém não foi encontrada nenhuma correlação, mas verificou-se a presença de outliers que poderiam distorcer a amostra. Então converteu-se a média das transferências por aluno em log, contudo, também não foi encontrada correlação entre a média de transferências por aluno e os coeficientes.
